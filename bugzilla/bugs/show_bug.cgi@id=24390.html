QuickFolders 2.8 (at the moment). Both newly installed since bug 24389.

At almost every startup, QuickFolders loads a web page with its version history
_and_ pops up an alert box telling me that it was successfully updated to some
version or other.

In an attempt to override this annoying behaviour and look for updates when it
suits _me_ (about daily, just before installing the next SeaMonkey nightly), I
have clicked "Reset all add-ons to update manually" in the add-ons manager, and
even "Check for updates: Never" in the details view for QuickFolders, to no
avail: the web page and alert box still appeared at the latest startup.

See bug 24389 for details.

This bug is about the fact that QF updates itself behind my back, without my
say-so and even against my direct orders; bug 24389 is about the annoying
webpage and popup which are displayed in that circumstance.
------- Comment #1 From Axel Grude 2011-09-11 10:40:20 ------- 
Hi Tony, I think you are seeing 2 bugs here.

1. the update was only done once, but for some reason it never finished. When
QF is updated completely it saves the version number in the setting

extensions.quickfolders.version

If there is no entry and you install QF it should bring you the Mozdev QF
homepage.

If there is an existing entry and it _differs_ from the current version it
should be updated to the new version and then bring you to the version +
donation pages.

QF does not look for updates on its own account, nor does it change its version
number when the Host version changes  (Unless this causes a completely new
version number to be downloaded).

The  behavior you are reporting points to an unfinished installation (due to a
bug in QF or another extension) that stops quickfolders from writing this
version number. Can you check Tools / Error Console of there are any errors
listed? I can also give you direct help with console (best to use console2) and
its settings if you email me directly!

First of all, can you please check what the current number of 

extensions.quickfolders.version

in about:config says? My current version is 

"2.8hc-AddonManagerVersionPending"

I had to bump up the version number from plain old 2.8 to last minute fix a bug
shortly before it was reviewed by AMO. Straight after I asked for an expedited
review of QF. So when the official version was released it was already the
(rather awkward) number above.

Also, on the next start of SeaMonkey can you enable Debug mode and right click
it and then also enable the setting 
extensions.quickfolders.debug.firstrun

(the easiest way to get at this setting is by right-clicking the debug box in
the options dialog.)
