tested on Tb 58 beta3. in my case the editor does focus to the top of the
document (but ignores my %cursor% variable in all cases)

Debugging is, I found that apparently the false re-focusing (or in your case
not focusing at all) is happening after SmartTemplate4 has processed the

NotifyComposeBodyReady

event. So it is due to some change within the Thunderbird code base, which must
do additional processing after the NotifyComposeBodyReeady event. I guess we
need to raise a bug with the Thunderbird bugzilla.
