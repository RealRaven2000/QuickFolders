Bugs + Feature Requests

Github - why, and how?

Github is free and open to anyone who registers with it. The big advantage here is that many users can share bug information, or view feature requests or planned new features and track these, in true open source fashion. From time to time we will also publish patched prerelease versions by attaching them to their relevant bugs.

When submitting new bug reports, please search first whether the bug is already submitted to avoid yours being marked as duplicate.

QuickFolders uses Github to maintain information on all bugs. It is relatively easy to submit a bug report or upload screen shots, you will have to create a free account with Github to do this. As with a lot of open source products, this is also a community driven experience, the more people contribute here, the better and more efficient work on future versions of QuickFolders will be.

For new feature requests, I often upload test versions to these issues - if you have created an issue or commented on one you will be automatically notified about news on this particular topic; this way you get to add your experience # and directly influence the creation of new functionality.

If you have found a bug it may be helpful to create an error log using Thunderbird's built in error console. See the troubleshooting section for details on this.
