------- Comment #1 From Tony Mechelynck 2011-09-02 11:11:06 ------- 
(In reply to comment #0)
Workaround: install the Add-on Compatibility Reporter extension to override
compatibility checking. This way, I think that Quick Folders should work even
in Thunderbird 9.0a1 (I haven't tested it in Thunderbird, but I use it in the
equivalent SeaMonkey 2.6a1).

Of course, this makes you responsible of disabling any misbehaving extensions:
the fact that they aren't "paperwork-compatible" will still be mentioned in the
Add-ons Manager but it won't cause auto-disabling anymore. The difference
between paperwork incompatibility and true code-level incompatibility is what
you should be attentive to.

This workaround is of course only a temporary device, not a real fix.
------- Comment #2 From Axel Grude 2011-09-02 12:02:36 ------- 
Didn't realize this until last Wednesday, and I had a beta in the works. I will
release the new version (2.7.3 or 2.8) this weekend. Fore the interim, I have
uploaded a compatible beta version  [usable with Tb 6.*] at
http://downloads.mozdev.org/quickfolders/

Alternatively you can use Tools / Addons Manager > Get Add-Ons > Install Addon
from File... and then paste this URI:
http://downloads.mozdev.org/quickfolders/QuickFolders-tb-pb-sm-2.7.3pre4.xpi

into the File field, and then click Open. 

Getting this ready took a little while as I had to iron out some problems with
the version checking algorithm which was completely changed by Mozilla since
version 4. If you experience anything unusual about the update (above and
beyond displaying the usual version history and donation page) please do get
back to me so I can fix it for everybody else. I have smoke-tested this
pre-release version with TB6.0.1, Postbox2.5 and SeaMonkey 2.2 under Windows
XP, so if find you bugs that might be relevant to other (mail application or
operating system) versions, also please report them back in this bug.

------- Comment #6 From Tony Mechelynck 2011-09-03 01:33:54 ------- 
(In reply to comment #4)
[...]
> Is there a 'Safe Mode' start-up for TB?

Yes.

There may or may not (depending on version) be a "Help â†’ Restart with Add-ons
Disabled" menuitem (the easiest if it exists), but even if there isn't you can
start a command prompt window (cmd.exe on Windows, konsole or gnome-terminal on
Linux, Terminal on mac), and run

    thunderbird -safe-mode

You may have to prefix the executable name by a full path if it is not in your
PATH.

In safe mode, all add-ons are listed as (disabled) in the add-ons manager, but
those which wouldn't be disabled out of thafe mode have a [ Disable ] button,
the others have [ Enable ]
