------- Comment #1 From Axel Grude 2016-01-21 15:12:17 ------- 
Let's get started with the most important bi8t: saving all Tabs and their
colors. They are currently stored in a single about:config setting

QuickFolders.Folders

and can grow to a quite impressive length, especially when you have loads of
customized folders; this can theoretically lead to performance problems when
starting Thunderbird or saving settings.

Currently it is possible to copy this information to the clipboard using the
"Folders?" button on the QuickFolders Options advanced tab - which creates a
"Beautified" easily readable JSON text, but unfortunately this currently cannot
be imported back in. So the goal is to turn this into a "backup" and "restore"
button first. Later on we can add the other (global) settings (such as folder
behavior) of QuickFolders as optional inclusion into the file.

In order to make it extensible I will change the current json format and wrap
the folder string into a wrapping object which I may call Entries or Model.

Example:

{
  Entries: [
    {
      "uri": "mailbox://nobody@smart%20mailboxes/Templates",
      "name": "All Templates",
      "category": "[Smart Folders]",
      "tabColor": 0
    },
    {
      "uri":
"mailbox://account@mailbox.com/Regs+Accounts/Politics/Dan%20Meyers",
      "name": "Dan Meyers",
      "category": "Friends",
      "tabColor": 0,
      "icon": null,
      "toAddress": "abcdefgh@gmail.com"
    },
    ...
  ] 
}
------- Comment #2 From Tony Mechelynck 2016-01-22 11:27:57 ------- 
This setting can of course be saved and restored manually in its "unbeautified"
state, in two ways:

a) via the clipboard, while the Thunderbird or SeaMonkey is running:
go to about:config (in Thunderbird, open the Config Editor in the Advanced
preferences), if necessary say that you'll be prudent, and enter QuickFolders
in the Filter box. You'll see QuickFolders.folders at the top. It can be saved
to the clipboard by right-click â†’ Copy Value, and restored later (after
having set the clipboard to the saved value) by double-click followed by Ctrl+V

b) via prefs.js
When the mailer is closed, the file prefs.js in its profile contains the saved
preference values from the latest session, one pref per line. Copy the relevant
lines into a text file: later, these same lines can be reinserted into prefs.js
while the mailer is not running. You can also set them in user.js (which does
not exist by default) instead of prefs.js, but with the following caveats:
user.js is only read at startup and never written (so the changes you make
there will not be overwritten), and it overrides any preferences which were
saved from a previous session (so if you forget you've used it you may be
surprised when you see that new changes to those prefs don't "stick" from one
session to another).
This second method allows saving all the "extensions.quickfolders.<something>"
preferences (if any of them were changed from their default) in addition to
QuickFolders.folders; since the file is saved sorted on the preference name it
is easy to avoid having two lines for the same pref. But you should avoid
modifying any lines concerning prefs you don't know about. (N.B. Only "user
set" prefs are saved at closedown; if a preference is omitted there it means it
has its default value.)

See also:
http://kb.mozillazine.org/Profile_folder_-_Thunderbird
http://kb.mozillazine.org/Profile_folder_-_SeaMonkey
http://kb.mozillazine.org/Prefs.js_file
http://kb.mozillazine.org/User.js_file
http://kb.mozillazine.org/Editing_configuration
